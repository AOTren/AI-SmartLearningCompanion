## 一、产品概述

### 1.1 背景与痛点

高校学生在期末复习阶段普遍面临以下问题：

| 痛点 | 具体表现 | 影响 |
|------|----------|------|
| **信息过载** | 课程课件 50+ 份，PPT/Word/PDF 格式混杂，内容碎片化 | 不知道从何看起，复习效率极低 |
| **无重点可循** | 整本教材或全套课件逐一翻阅，缺乏考点优先级排序 | 时间有限却花在非重点上 |
| **多模态解析难** | PPT 中包含 ER 图、流程图、公式等，纯文本检索无法覆盖 | 关键知识点被遗漏 |
| **被动式学习** | 传统方式"看书→做题→对答案"，缺少主动引导和错因追踪 | 做了也不懂为什么错，下次还错 |

### 1.2 产品定位

**AI 期末私教** 是一款基于 RAG（检索增强生成）技术的智能伴学平台，面向高校学生的期末复习场景，将碎片化的课程课件转化为结构化知识库，通过 **"考点提问→真题演练→错因分析→举一反三"** 的四步闭环工作流，提供个性化、有重点的 AI 辅导体验。

### 1.3 核心价值主张

> **把 50+ 份杂乱课件，变成一个懂考点、会出题、能纠错的 24 小时私教。**

## 二、用户画像

### 2.1 目标用户

| 维度 | 描述 |
|------|------|
| **人群** | 高校本科生（以理工科为主，如数据库、数据结构等课程） |
| **场景** | 期末考前 2~4 周，集中复习阶段 |
| **设备** | 以笔记本电脑为主（网页端），辅以平板浏览器 |
| **技术水平** | 普通数字原住民，会用基本搜索，不熟悉 AI 工具细节 |
| **期望** | "告诉我要考什么，帮我练，告诉我哪里错了，再给我类似的题" |

### 2.2 用户故事

```
作为【一名正在准备数据库期末考试的学生】
我希望【输入课程章节名就能知道这章的重点考点】
以便【不再逐页翻阅 200+ 页的PPT】
```

```
作为【一个刚做完一道真题但不确定对不对的学生】
我希望【系统不仅能判断对错还能分析我的错误原因】
以便【理解背后的原理而不只是记住答案】
```

```
作为【一个觉得某个知识点还没掌握的学生】
我希望【系统能给我出几道同类型的练习题】
以便【举一反三彻底掌握这个考点】
```

### 3.2 功能详细说明

#### 3.2.1 智能对话模块

| 功能点 | 说明 |
|--------|------|
| **消息发送** | 文本输入框 + 回车/按钮发送 |
| **流式接收** | 通过 SSE 接收后端转发的 Coze 流式响应，逐字/逐段渲染 |
| **消息气泡** | 用户消息靠右（蓝色）、AI 消息靠左（白色/灰色卡片） |
| **加载状态** | 发送中显示打字动画，等待回复时显示骨架屏/脉冲动画 |
| **Markdown 渲染** | 支持标题、列表、粗体、代码块、表格等 Markdown 格式 |
| **数学公式** | 支持 LaTeX 公式渲染（行内 `$...$` 和独立 `$$...$$`） |

#### 3.2.2 四步伴学工作流

这是产品的**核心差异化功能**，前端需要提供对应的快捷操作入口：

**步骤一：考点提问**

| 项目 | 说明 |
|------|------|
| **触发入口** | 侧栏「📝 考点提问」按钮 / 对话中自动引导 |
| **用户行为** | 选择或输入章节名 → 系统返回该章考点梳理 |
| **预期输出** | 结构化考点列表（含重要级标记 ★/★★/★★★）、关联知识点说明 |
| **UI 表现** | 考点以卡片形式展示，每条含简要描述 |

**步骤二：真题演练**

| 项目 | 说明 |
|------|------|
| **触发入口** | 侧栏「📝 真题演练」按钮 / AI 回复底部快捷操作卡 |
| **用户行为** | 系统推送一道选择题/填空题 → 用户点击选择或输入答案 |
| **前端交互** | 选项以按钮组呈现，点击后立即显示 ✅/❌ 反馈 |
| **联动逻辑** | 答对→提示进入下一步；答错→自动触发「错因分析」 |

**步骤三：错因分析**

| 项目 | 说明 |
|------|------|
| **触发入口** | 答错后自动触发 / 侧栏「❓ 错因分析」按钮 / AI 回复底部快捷操作卡 |
| **用户行为** | 无需额外操作，系统自动输出分析 |
| **预期输出** | 错误原因归类（概念混淆/计算失误/审题不清等）、正确思路推导、对应教材原文引用 |
| **UI 表现** | 分析内容以醒目样式呈现，配合引用来源卡片 |

**步骤四：举一反三**

| 项目 | 说明 |
|------|------|
| **触发入口** | 侧栏「🔁 举一反三」按钮 / AI 回复底部快捷操作卡 |
| **用户行为** | 系统基于当前考点生成 2~3 道同类变式题 |
| **预期输出** | 同考点不同角度的题目，帮助巩固 |
| **循环** | 完成后又可回到步骤二，形成学习闭环 |

#### 3.2.3 引用溯源模块

Coze RAG 返回的内容中包含知识库来源引用，前端需做专门展示：

```
┌─────────────────────────────────────┐
│  📎 引用来源                         │
│  ┌────────────────────────────────┐ │
│  │ 📄 第三章-关系数据库理论.ppt      │ │
│  │    第 12 页 · Slide: "范式定义"  │ │
│  │    匹配度: 94%                  │ │
│  └────────────────────────────────┘ │
│  ┌────────────────────────────────┐ │
│  │ 📄 2024年期末试卷及答案.pdf       │ │
│  │    第 3 题 · 大题(2)            │ │
│  │    匹配度: 87%                  │ │
│  └────────────────────────────────┘ │
└─────────────────────────────────────┘
```

| 规则 | 说明 |
|------|------|
| **出现时机** | AI 回复中涉及具体知识点时，紧跟在相关段落后方 |
| **可折叠** | 默认展开，支持收起，不占用过多视觉空间 |
| **置信度颜色** | ≥90% 绿色 / 75%~89% 黄色 / <75% 橙色 |
| **数量上限** | 单次回复最多展示 5 条引用 |

#### 3.2.4 练习交互组件

针对选择题的专用 UI 组件：

| 状态 | 视觉表现 |
|------|----------|
| **默认** | 白色背景，灰色边框，悬停时边框变色 |
| **已选（未提交）** | 蓝色边框 + 浅蓝背景 |
| **正确** | 绿色背景 ✅ + 正确答案解析淡入 |
| **错误** | 红色背景 ❌ + 正确答案高亮 + 「查看错因」按钮浮现 |

#### 3.2.5 历史记录模块

| 功能 | 说明 |
|------|------|
| **自动保存** | 每轮对话结束后自动写入 localStorage |
| **会话列表** | 点击历史图标展开列表，显示时间戳 + 首句摘要 |
| **恢复会话** | 点击某条历史记录可恢复该次完整对话 |
| **导出** | 支持将当前对话导出为 Markdown 文件下载 |
| **清空** | 一键清空所有本地历史（二次确认） |

#### 3.2.6 界面定制

| 功能 | 说明 |
|------|------|
| **主题切换** | 顶栏提供明/暗模式切换按钮，使用 CSS 变量实现 |
| **偏好持久化** | 主题选择存入 localStorage，刷新不丢失 |
| **默认主题** | 首次访问跟随系统偏好（`prefers-color-scheme`） |
| **侧栏控制** | 侧栏可完全收起，给对话区更大空间 |

### 4.2 核心页面/状态描述

#### 状态 A：首次进入（欢迎态）

- 侧栏展开，显示章节目录
- 对话区显示**欢迎语卡片**：
  - 问候语 + 产品简介（一句话）
  - **3 个推荐操作快捷卡片**：「📖 开始复习第 X 章」「📝 来一道真题」「❓ 我不太清楚怎么用」
- 底部步骤指示器高亮步骤 ①

#### 状态 B：对话进行中

- 消息气泡正常展示
- AI 回复期间：**流式文字逐段出现** + 打字光标动画
- AI 回复完成后：底部浮现**快捷操作卡**（根据上下文动态变化）

#### 状态 C：练习模式

- AI 消息中嵌入**练习题卡片**
- 选项以交互按钮呈现
- 用户点击选项后触发状态变更动画
- 结果反馈后浮现下一步操作引导

#### 状态 D：历史记录弹层

- 右侧滑出抽屉面板
- 列表形式展示历史会话
- 每项含：时间、首句摘要预览、章节点标签

### 4.3 关键交互流程图

#### 流程 1：完整四步伴学流程

```
[用户] 选择章节（如"第三章 关系数据库理论"）
  ↓
[系统] 返回该章考点清单（带引用来源）
  ↓
[用户] 点击「来一道真题」
  ↓
[系统] 推送一道选择题 + 4个选项按钮
  ↓
[用户] 点击选项 B
  ↓
  ├─ 正确 → 显示 ✅ + 解析 + 「再来一道」「举一反三」
  │
  └─ 错误 → 显示 ❌ + 高亮正确答案 + 自动触发错因分析
           ↓
       [系统] 输出错因分析（错误归类 + 思路纠正 + 教材引用）
           ↓
       [用户] 点击「举一反三」
           ↓
       [系统] 生成 2~3 道同类变式题
           ↓ （回到练习环节，循环）
```

#### 流程 2：自由问答流程

```
[用户] 在输入框输入任意问题（如"范式是什么意思？"）
  ↓
[系统] 通过后端调用 Coze API → 流式返回回答
  ↓
[前端] 逐字渲染 Markdown + 公式 + 引用来源卡片
  ↓
[完成] 底部显示快捷操作卡
```

## 五、非功能性需求

### 5.1 性能要求

| 指标 | 要求 |
|------|------|
| **首屏加载时间** | ≤ 2 秒（不含 API 调用） |
| **流式首字节延迟** | ≤ 500ms（从发送到收到第一个 token） |
| **UI 操作响应** | ≤ 100ms（界面交互如切换主题、打开侧栏） |
| **单页内存占用** | ≤ 80MB（长时间使用不泄漏） |

### 5.2 兼容性要求

| 维度 | 要求 |
|------|------|
| **浏览器** | Chrome 90+、Edge 90+、Firefox 88+、Safari 14+ |
| **分辨率** | 最小支持 375px（移动端），最优 1280×800+ |
| **字体回退** | 中文：苹方/微软雅黑/黑体；英文/代码：SF Mono/Consolas/Monaco |

### 5.3 可用性要求

| 规则 | 说明 |
|------|------|
| **离线降级** | 网络断开时提示重连，不丢失已加载内容 |
| **错误友好** | API 报错时显示中文友好提示，不暴露技术细节 |
| **键盘支持** | Enter 发送、Shift+Enter 换行、Esc 关闭弹窗 |
| **无障碍** | 图片加 alt、按钮有 aria-label、色彩对比度 ≥ 4.5:1 |

### 5.4 数据安全要求

| 规则 | 说明 |
|------|------|
| **API Key 保护** | Key 仅存储在后端 `.env`，前端不可见 |
| **传输安全** | 生产环境强制 HTTPS |
| **本地存储** | 对话历史仅存 localStorage，不上传服务器 |
| **敏感词** | 前端不做过滤（由 Coze 端把控） |

## 六、技术方案概要

### 6.1 前端技术栈

| 类别 | 技术 | 用途 |
|------|------|------|
| **基础** | HTML5 + CSS3 + ES6+ | 页面结构与样式 |
| **构建** | 无（原生开发） | 零依赖启动，降低复杂度 |
| **Markdown** | marked.js (CDN) | AI 回复内容渲染 |
| **公式** | KaTeX (CDN) | 数学公式渲染 |
| **代码高亮** | highlight.js (CDN) | 代码块语法着色 |
| **通信** | Fetch API + SSE | 与后端通信，流式接收 |
| **存储** | localStorage | 会话历史本地持久化 |

### 6.2 后端中间层技术栈

| 类别 | 技术 | 用途 |
|------|------|------|
| **框架** | Python FastAPI | 异步 Web 服务 + SSE 支持 |
| **HTTP 客户端** | httpx (async) | 调用 Coze Open API |
| **配置管理** | python-dotenv | 环境变量加载 |
| **CORS** | fastapi.middleware | 跨域请求处理 |

### 6.3 Coze API 集成

| 项目 | 说明 |
|------|------|
| **接口** | `POST https://api.coze.cn/v3/chat` |
| **认证** | `Authorization: Bearer {PAT}` |
| **流式** | `stream: true` → 返回 SSE 事件流 |
| **关键参数** | `bot_id`、`user_id`、`messages`、`chat_history` |
